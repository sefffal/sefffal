Octofitter

Octofitter is a large set of packages I created with the goal of advancing the current state of orbit fitting. Octofitter is built from the ground-up for extremely high performance and differentiable computing. Using Octofitter, challenging orbit models which previously would require hours or days, can now be sampled in seconds or minutes.

This speed enables researchers to approach their modelling tasks with a new level of rigour, and following a full Bayesian workflow. One can—in less than the time it used to take to run a single model—compare how different individual data points impact a fit, compare how priors affect the results, or even run full suites of simulated fits to anticipate how new data would impact findings.

Octofitter supports an even-growing list of kinds of exoplanet data. At present, these include:

  • Relative astrometry
  • Barycentric radial velocity (e.g. of a star)
  • Barycentric radial velocity (e.g. of a star)
  • Hipparcos-Gaia proper motion anomaly
  • Hipparcos IAD
  • Relative photometry
  • Absolute photometry
  • Direct images
  • Interferometric closure phases and squared visibilities e.g. from sparse aperture masking
  • GRAVITY closure phase data using kernel phase analysis
  • Arbitrary x vs. y likelihood maps e.g. from GRAVITY, spectral cross correlation, etc.
  • Some spectral modelling features, e.g. to support GRAVITY data
In addition, all of the above data can be combined together through a user-specified joint model. These models can freely incorporate the following features:
  • Tie dynamical masses and photometry together through age & cooling tracks
  • Specify custom priors, including multivariate priors with correlations, etc.
  • Specify prior distributions using kernel density estimates (just pass raw samples).
  • Freely transform variables to different bases
